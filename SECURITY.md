# Security Policy

This security policy applies to public projects under the [mattriffle][gh-account] account on 
GitHub.

## Security/Bugfix Versions

Security and bug fixes are generally provided for the last minor version.  Fixes are released either 
as part of the next minor version or as a patch version.

Security fixes are given priority based on severity. 

## Reporting a Vulnerability

We encourage responsible disclosure of security vulnerabilities. Thank you for your consideration.

### Ways to report

The preferred way to report is via the "Report a vulnerability" button under the "Security and
quality" tab of the associated GitHub project.  This creates a private communication channel.

If you are unable to use GitHub's vulnerability reporting workflow, please reach out via email to
matt (at) mattriffle.com.

[gh-account]: https://github.com/mattriffle
